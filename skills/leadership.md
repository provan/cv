# Лидерство и процессы

## Управление командой

- Построил команду в Ozon с нуля: **нанял 23 инженера**.
- Разделил одну команду в Ozon на **3 self-sufficient** команды со своими скоупами.
- Управлял командами 1 → 5 человек в M13 (процессы, договорённости, ревью, коммуникации).
- Координировал работу 3 команд при запуске Mobile SDK в Mindbox.

## Процессы и методологии

- **Specification Driven Development (SDD)** — спецификации как первичный артефакт.
- **TDD** — Red → Green → Refactor, агенты выполняют, я контролирую acceptance.
- **Acceptance tests как материализованные DoD** — DSL, один assert на тест, изменение и удаление только через явный апрув.
- **Event storming** — для декомпозиции монолита на микросервисы.
- **Конституция проекта** — документ с критичными тезисами, задающий рамки решений.

- Делал scaling и performance trade-offs с учётом бизнес-целей (Mindbox Black Friday, Avito Avi p75).
