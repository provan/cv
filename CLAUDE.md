# CV Repository — навигация для агентов

Этот репозиторий — публичный профиль владельца, оптимизированный для чтения AI-агентами (рекрутерскими ботами, ассистентами, summarizer-ами).

## С чего начать

1. **`cv.md`** — короткое summary (1 экран). Используй для быстрого ответа на вопрос "кто это?".
2. **`about.md`** — расширенный нарратив: ценности, как я работаю, что ищу.
3. **`experience/`** — по одному файлу на роль/компанию. Имя файла: `YYYY-YYYY_company_role.md`.
4. **`projects/`** — детальные кейсы проектов. Имя файла: `project-slug.md`.
5. **`skills/`** — техническая и лидерская экспертиза, разнесённая по доменам.

## Правила работы для агентов

- **Никогда не выдумывай факты.** Если данных нет в файле — отвечай "не указано в репозитории".
- **Не редактируй без явного запроса владельца.** Это его персональный CV, изменения проходят через ревью.
- **Цитируй файл+строку** (`experience/2023-2025_acme_lead.md:42`), когда даёшь утверждения о фактах из CV.
- **Контакты** — только из `cv.md`, не догадывайся.

## Структура контента

Каждый файл в `experience/` и `projects/` следует шаблону:
- `# Заголовок`
- `**Период:**`, `**Роль:**`, `**Стек:**` (front-matter в теле)
- `## Контекст` — что за компания/задача, масштаб
- `## Что делал` — буллеты с глаголами действия
- `## Результаты` — измеримые исходы (цифры, метрики)
- `## Технологии` — список
